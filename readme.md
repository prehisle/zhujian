# 朱笺(zhujian)

个人事务管理工具 —— **纯人工、灵活不受限、维护性高**。名取「朱砂笺纸」:古人拿朱色小笺写短札记小事,正合本品的纸与朱墨设计(项目原名 ys-notebook)。

```
快速捕获 → 灵感沉淀 → 手动流转(打标签 / 转待办)→ 任务看板推进
```

单用户自用的本地优先(local-first)桌面工具。捕获低摩擦、原文永不丢失、**全程不需要 AI 参与**:想法的每一步流转都靠手动操作完成,数据始终在本机 SQLite。

> **定位变迁**:本工具早期是「AI 想法流转系统」,2026-06-26 起重新定位为纯人工工具——AI(整理/补提)代码已**物理删除**,界面与数据模型都做了减法。**2026-06-28 ㉜ 进一步把想法与待办收成单实体**(同一行翻状态、零副本)。要再用 AI(决策辅助/总结)需从头加。

## 技术栈

| 层 | 选型 |
|---|---|
| 桌面外壳 | Tauri v2(Rust 后端 + WebView 前端) |
| 唤起方式 | 全局热键 `Ctrl+Alt+N`(记录灵感)/ `Ctrl+Alt+M`(打开主窗,停在你上次的视图)+ 系统托盘(**双击=打开主窗,右键=菜单**,菜单显示对应快捷键) |
| 前端 | Vite + TypeScript(原生,无框架) |
| 存储 | SQLite(rusqlite,bundled) |

## 功能

- **快速捕获**:全局热键 / 托盘唤起一个浮窗,输入灵感 → 回车存入「灵感」列表 → 窗口闪退。**可直接粘贴图片**(Ctrl+V):粘的图先在浮窗里显小预览(可单独移除),回车时随这条灵感一起存(图带「图N」编号);只贴图不打字也能存。
- **灵感**(笔记本主窗·默认视图):两个 Tab——
  - **灵感**:顶部「新建灵感」compose(**也能 Ctrl+V 粘图**,预览暂存、回车随灵感一起存,同捕获浮窗);打过标签和没打标签的灵感**在同一个列表里**(标签只是顺手的元数据,不再拆成两个 Tab),并**按天分组成时间轴**(今天/昨天/前天/日期标头 + 左侧竖轴,卡片显时分)。灵感多了顶部还有**标签筛选 + 文本过滤**(所有 / 无标签 / 各标签,和看板同一套,两者可叠加;文本词清空或 `Esc` 即回,标签切回「所有」即回;筛着某个标签「记下」的新灵感**自动挂上该标签**,文本过滤着「记下」则自动清词——新卡不会被自己滤没)。每条灵感可【编辑(保留历史)】【配图】【转待办】【标签】【复制】【删除】;打了标签的就长出标签 chip;【删除】一律**软删进回收站**(可还原,所以不弹确认——手滑了随时找回来),真要销毁去回收站【彻底删除】。(注:**转待办后灵感变成任务、离开灵感去看板**——同一主体换了档位,不再留在这里。)头部还有一行淡字**流转统计**:`本周捕获 N · 转待办 X%`(生而为灵感的条目里有多少后来转成了待办;数字从启用这项统计起积累,更早的旧数据不猜不算)。
  - **回收站**:软删的想法可【还原】/【彻底删除(二次确认)】/【清空】。
- **任务看板**:四列 **待办 / 进行中 / 待确认 / 已完成**——
  - **拖动**在列间改状态、列内手动排序(拖到落点位置插入);把「已完成」拖到底部归档区**入成就册**(真归档,不是删除;归档区平时不占眼,拖起已完成的卡时才在底部现身)。
  - **待确认**(可选第四列):事情做完但在等外部确认/回音的任务拖到这一列,从「进行中」分离开;**非强制**——可「进行中→已完成」直接跳过,四态自由双向流转(可打回重做)。
  - 卡片可设**截止日期 / 优先级 / 标签**,逾期/今天到期高亮;可【编辑改名】【配图】【复制】【删除(软删进回收站,首次二次确认 + 不再提示)】。
  - **悬停即选中 + 快捷键速查**:鼠标移到卡片,右上角 ⋯ 弹出操作速查(每项标着单键),也可直接按键直达——编辑 `E` / 复制 `C` / 标签 `L` / 截止 `S` / 优先级 `P` / 撤回 `B` / 删除 `D` / `]` 后移一列 / `[` 前移一列(键盘也能在四列间挪卡,不必拖动)。**同一套悬停 + 快捷键交互灵感卡也用**(重叠操作键义一致 E/C/L/D,便于养成单键操作)。**双击卡片**(空白处)= 默认操作「编辑」,灵感卡与任务卡都通。
  - **待办**任务可【撤回为灵感】(灵感=没想清楚的任务,同一主体退回更早的阶段):**同一条**翻回灵感——还挂着标签就落「已归类」、什么标签都没有才回「未归类」;**进行中/已完成**已是想清楚的任务,不给此钮。
  - 顶部**按标签筛选**(所有 / 无标签 / 各标签)+ **文本过滤框**(任务多了找卡难:输入即筛,只匹配任务标题;和标签筛选可叠加,清空或 `Esc` 全部回来。新建任务时自动清空过滤——新卡不会被自己滤没。要跨视图、连历史旧版一起找,用左侧「搜索」),筛选态下也能拖动;header 可「复制看板」为 Markdown。
  - **归档(成就册)**:干完的活是历史成就,不该和垃圾挤一个桶——「已完成」列头一键**全部归档**(两步确认),或拖到底部归档区、或单卡菜单【归档】;归档视图按**归档日分组成时间轴**,头部一行**统计**(`本周归档 N · 累计 N`),可查、**不可删**(想删先【取消归档】回看板,再走正常删除——多一步,防冲动)。
  - **回收站**视图:删除的任务可还原(回原列)/彻底删除/清空。归档 ≠ 删除,两边各走各的门。
- **标签**:轻量分类。每个标签一行显示 `N 条灵感 · M 个任务`;**点标签行就地「收缩展开」**(▸/▾)看挂该标签的灵感 + 任务(只读,任务带所在列 + 截止/优先级)——展开/收起在原地,不跳到单独页。人工 **新建 / 重命名 / 删除 / 合并**(把碎标签并成一个,重写归属投影)。**想分层就用斜杠命名**:建个 `zhujian/同步`,只要同名的 `zhujian` 标签也存在,它就在标签列表里缩进到 `zhujian` 下面、只显示「同步」(悬停见全名)——**只是看着有层级**,筛选/计数/合并都还把它当独立标签(筛 `zhujian` 不会自动包含 `zhujian/同步`;一条内容两边都想算,就两个标签都打上)。
- **全局搜索**:按内容跨「灵感 / 任务 / 回收站」查找任意条目,**连改过的旧版本一起搜**(归档的成就也搜得到,徽标显「归档」),匹配高亮,带状态徽标 + 标签 chip 作定位。
- **配图**:纯文字说不清时给灵感 / 任务配图。**凡是能输入正文的地方都能粘**——编辑卡片、捕获浮窗、灵感/看板的新建输入框,**粘贴截图(Ctrl+V)**即可挂上(新建时先显暂存预览、保存随条目一起存),一条可配**多张**;每张缩略图带**「图N」编号**角标,点开看大图,正文里写「图1」会自动变成可点链接跳到那张图。编号是图的「身份证」——删掉某张图后剩下的不重新编号(留洞),正文引用永远指向同一张图;换图 = 删旧加新。图片随条目一起进回收站 / 还原 / 彻底删除。
- **正文链接**:灵感 / 任务正文里的 `http(s)://` 网址自动变成可点链接——**左键点**用系统默认浏览器打开,**右键点**复制链接(想粘到指定浏览器时用),悬停看完整地址。
- **编辑即存**:双击卡片(或 `E`)进编辑,**回车 / 点到别处**都保存,`Esc` 取消——不用找保存按钮。
- **记住窗口和视图**:主窗的大小 / 位置 / 最大化状态,以及上次停在哪个视图(灵感 / 任务 / 标签 / 搜索),退出重启后原样回来;首次运行用默认大小居中。
- **折叠侧栏**:侧栏顶角小按钮或 `Ctrl+B` 把左侧栏收成细条,给内容腾出横向空间;再点 / 再按展开,折叠状态记住。
- **多设备同步(试验中,自建服务器)**:侧栏底部「同步」入口——多台设备经自己的服务器同步数据,**端到端加密**(笔记与图片加密后才出本机,服务器只见密文、只做转发和暂存)。第一台设备点「创建账户」即可(无需邀请码),会得到一串**恢复码**(抄在纸上:它是账户密钥、不是数据备份——恢复数据还必须有至少一台在线的完整副本,服务器帮不了);其余设备输入老设备生成的**配对码**加入,自动拿到全量数据,此后离线照用、上线互补。未配置时这个功能完全沉默,不打扰单机使用。
- **空间**:侧栏顶部当前空间名(默认「默认空间」)点开可**新建空间**(比如「家庭」,数量不限)并随时切换、改名;手机端同样支持多空间。**空间有两条来路**:「新建空间」=开个新本子,即建即用、纯本地(要多端同步,在那个空间里创建账户);「加入空间」=把别处已有的账户接到本机(扫码/输配对码),同步完成才出现在列表里。每个空间是一本**完全独立的册子**——各自的灵感/看板/标签/搜索、各自的同步账户;把家人设备配对进「家庭」空间的账户,两个人就共用这一本,而**默认空间的数据永远不出它自己的账户**(隔离是加密边界,不是权限开关)。快速捕获落**当前所在空间**(浮窗右上有小印标明落点,只有一个空间时不显);跨空间不混排:搜索/看板只看当前空间,切换着看。
- **移动到其他空间**:记错了地方,灵感卡/看板卡的 ⋯ 菜单里「移动」(快捷键 M)可把这一条搬进别的空间(有两个以上空间时才出现)。移动 = 在目标空间重新诞生、在原空间删除:标签按名字带过去、配图原编号带过去;**编辑历史不随迁、会随移动永久删除**(选择空间前会提示)。万一中途出岔子,宁可两边各留一份也绝不弄丢,卡片上会明确标注让你手动处理。

## 开发

前置:Node、Rust(rustup stable-msvc)、WebView2 运行时、VS2022 C++ 工具集。

```bash
npm install          # 前端依赖(含 Tauri CLI)
npm run tauri dev    # 启动开发模式(同时起 vite:1420 + app)
npm run tauri build  # 打包发行版(release,把前端嵌入独立 exe)
```

> ⚠️ **别直接运行 `src-tauri/target/debug/app.exe`**:debug 版的 WebView 指向开发服务器
> `localhost:1420`,未先起 vite 会显示「localhost 拒绝连接」。开发请用 `tauri dev`,
> 需要独立可执行文件请用 `tauri build`。
>
> 应用是托盘程序,两窗口默认 `visible:false`——从托盘双击(或右键菜单「打开朱笺」)、热键 `Ctrl+Alt+N`(捕获)或 `Ctrl+Alt+M`(打开主窗)进入;主窗打开后停在你上次离开的视图。

## 数据模型要点

- 主键全为 **ULID**(TEXT),时间戳为 RFC3339 UTC TEXT;`due_on` 例外,是用户本地日历日 `YYYY-MM-DD`(截止是「日历日意图」,不存 UTC instant 以免差一天)。
- **单实体(2026-06-28 ㉜)**:想法与待办是**同一行 `items` 的不同 `stage`**(`inbox 未归类 / filed 已归类 | todo / doing / confirming / done`)——不是两条记录。**转待办 = 翻 stage 到 todo(零副本)**,撤回 = 翻回 inbox/filed;任务态条目不出现在灵感视图(天然去重)。`archived_at` 是回收站轴(冻结 stage、还原回原档),前端按冻结 stage 分「灵感回收站 / 任务回收站」。
- **概念隔离**:`items.stage`=流程档位(灵感↔任务)/ `topics`+`item_topic`(M:N)=标签(对用户即「标签」;**㉞ 任务已是真多标签**——一个任务可挂多个标签,筛选时在每个所属标签下都出现)。
- **不可变性是历史级**:用户**可以编辑**自己的条目,每次改动前的旧版本由 DB 触发器(migration 0014 `trg_item_archive_on_edit`)自动归档进 `item_revisions`(append-only、不可改写、不可绕过),**覆盖全 stage**(改任务标题也归档),原文与每一版永不丢失。
- **删除是用户主权,但分级 fail-safe**:防的是「意外/静默毁数据」,不是禁用户清理自己的数据。
  - **删除=进回收站,销毁只在回收站里发生**(2026-07-08 起,未归类的灵感也一样):【删除】都是软删(盖 `archived_at`、可还原回原档)→ 回收站二次确认才彻底删(cascade 标签与历史)。灵感与任务**共用这一条 archived_at 轴**。存储层删除守护(migration 0014)照旧:已归类的想法/任务在数据库层面就硬删不掉活跃行;未归类灵感的硬删原语仍在,但只供命令层/测试清库用。
- **撤回为灵感**(灵感=没想清楚的任务,**仅限「待办」**回退,进行中/已完成不可):**同一行翻 stage 回灵感**——还挂标签则落「已归类」、什么都没有才回「未归类」;不再有「还原源想法 / 按标题新建」之分(本就是同一条)。
- **`topics`(标签)是可重算派生投影**:手动合并(`merge_topics`)重写 `item_topic` 归属(单次同时覆盖想法+任务的标签)、把碎标签并入存续标签、不保留 membership 历史——不丢 item、不碰 content。
- 每连接 `PRAGMA foreign_keys = ON`;关联表 `ON DELETE CASCADE`。不加 metadata JSON 万能列;扩展靠显式迁移。

## 设计铁律

- **纯人工**:任何想法流转(打标签 / 转待办 / 撤回 / 删除)都不依赖 AI,全部是手动事务。
- **fail-fast**:产品代码绝不写静默回退/兜底默认值;非法操作整事务回滚并报错。
- **只保「数据不丢失」类保证**:历史归档(0003)、禁直删 processed(0004)保留;早期为防 AI/系统静默改数据而设的「流转锁死」类触发器(suggested 闸门、归档状态冻结)随 AI 一起撤掉,换取灵活与低维护成本。
- **风格**:轻量 / 最小化 / 纸与朱墨(暖纸面 + 浓墨 + 朱砂点缀;想法原文用衬线,界面文字用黑体)。
